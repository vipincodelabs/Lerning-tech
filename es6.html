01

Template Literals

1

Definition

Template literals are string literals that use backtick (`) delimiters instead of quotes. They support multi-line strings, expression interpolation via ${}, and tagged templates for advanced string processing.

2

Why It Exists

Before ES6, building dynamic strings was error-prone:

JSBefore ES6
var name = "Vipin";
var msg = "Hello, " + name + "! You have " + (count + 1) + " messages.";
// Multi-line required ugly \n or string concatenation
var html = "<div>" +
           "  <p>" + text + "</p>" +
           "</div>";
3

Syntax & Core Concept

JSES6
const name = "Vipin";
const count = 5;

// 1. Basic interpolation
const msg = `Hello, ${name}! You have ${count + 1} messages.`;

// 2. Multi-line — newlines preserved literally
const html = `
  <div>
    <p>${text}</p>
  </div>
`;

// 3. Expression inside ${} — any valid JS expression
const label = `Status: ${isActive ? "Active" : "Inactive"}`;

// 4. Nested template literals
const items = ["a", "b"];
const list = `<ul>${items.map(i => `<li>${i}</li>`).join("")}</ul>`;
4

Practical Examples

Simple: URL construction

JS
const userId = 42;
const url = `https://api.example.com/users/${userId}/posts`;

Real-world — React JSX utility:

JSXReact
// Dynamic class names
const Button = ({ variant, disabled, children }) => (
  <button className={`btn btn--${variant} ${disabled ? "btn--disabled" : ""}`}>
    {children}
  </button>
);

// SQL query in Node.js (parameterized via tagged template)
const query = `
  SELECT * FROM users
  WHERE id = ${userId}
  AND active = true
`;

Tagged Templates — advanced use:

JSTagged Template
// A tag function receives (strings, ...values)
function highlight(strings, ...values) {
  return strings.reduce((result, str, i) =>
    result + str + (values[i] !== undefined ? `<mark>${values[i]}</mark>` : ""), ""
  );
}

const name = "Vipin";
highlight`Hello ${name}, welcome!`;
// → "Hello <mark>Vipin</mark>, welcome!"

// Real-world: styled-components uses tagged templates
const Box = styled.div`
  background: ${props => props.bg};
  padding: 16px;
`;
5

Common Mistakes

Warning
  • Using backticks everywhere "because it's modern" — plain quotes are fine for static strings (minor perf overhead from parsing)
  • Interpolating raw user input into SQL queries — SQL injection risk. Always use parameterized queries
  • Forgetting that multi-line backtick strings include indentation whitespace in the final string
  • Calling a tag function with parentheses: html(`...`) — the tag syntax is html`...` without parens
6

Advanced Insights

How Tagged Templates Work Internally

The engine splits the string at each ${} into a frozen array of string parts (strings.raw for raw backslash access) plus a values array. The tag function receives both and assembles the result — this is how styled-components and graphql-tag achieve zero-runtime overhead.

02

Destructuring

1

Definition

Destructuring is a syntax that lets you unpack values from arrays or properties from objects into distinct variables in a single, readable statement.

2

Why It Exists

JSBefore ES6
var user = { name: "Vipin", age: 25, role: "dev" };
var name = user.name;
var age  = user.age;
var role = user.role;  // repetitive, verbose
3

Syntax & Core Concept

Object Destructuring

JS
const user = { name: "Vipin", age: 25, role: "dev" };

// Basic
const { name, age } = user;

// Rename while destructuring
const { name: userName, role: userRole } = user;

// Default values
const { name, country = "India" } = user;

// Nested object
const config = { db: { host: "localhost", port: 5432 } };
const { db: { host, port } } = config;

// Rest in object destructuring
const { name: n, ...rest } = user;  // rest = { age: 25, role: "dev" }

Array Destructuring

JS
const colors = ["red", "green", "blue"];

// Basic
const [first, second] = colors;

// Skip elements with commas
const [, , third] = colors;  // third = "blue"

// Default values
const [a, b, c, d = "white"] = colors;

// Swap variables — classic trick
let x = 1, y = 2;
[x, y] = [y, x];

// Rest
const [head, ...tail] = colors;  // tail = ["green","blue"]
4

Practical Examples

JSXReact Hook
// useState returns [state, setter] — array destructuring
const [count, setCount] = useState(0);

// Function parameter destructuring
function UserCard({ name, avatar, role = "member" }) {
  return <div>{name} — {role}</div>;
}

// API response
const { data: users, error, isLoading } = useQuery("/api/users");

// Next.js page props
export default function Page({ params, searchParams }) {
  const { id } = params;
}
5

Common Mistakes

Pitfalls
  • Destructuring null or undefined throws a TypeError — guard first with user ?? {}
  • Confusing rename syntax: { name: newName } — the left is the key, right is the new variable
  • Deep nesting makes code unreadable — only go 2 levels deep max
  • Default value only applies when the value is undefined, not null
JSBug
const { role = "user" } = { role: null };
console.log(role); // null — default NOT applied for null!
6

Advanced Insights

Performance Note

Destructuring is syntactic sugar — V8 optimizes it to the same bytecode as property access. In hot loops over millions of iterations, avoid re-destructuring the same object — extract once before the loop.

03

Default Parameters

1

Definition

Default parameters let you specify fallback values for function arguments directly in the function signature. The default is used when the argument is undefined (or omitted).

2

Why It Exists

JSBefore ES6
function greet(name, greeting) {
  name = name || "Guest";         // Bug: falsy values like 0 or "" break this
  greeting = greeting || "Hello";
}
3

Syntax & Core Concept

JS
// Basic default
function greet(name = "Guest", greeting = "Hello") {
  return `${greeting}, ${name}!`;
}

// Default can be any expression — even a function call
function createId(id = crypto.randomUUID()) {
  return id;
}

// Later params can reference earlier params
function rect(width = 10, height = width) {
  return width * height;
}

// Works great with destructuring
function connect({ host = "localhost", port = 3000 } = {}) {
  return `${host}:${port}`;
}
Key rule

Default is applied only when the value is strictly undefined. Passing null, 0, or "" does NOT trigger the default.

4

Practical Examples

JSReal World
// API fetch wrapper
async function fetchData(url, options = {}, retries = 3) {
  const { method = "GET", headers = {} } = options;
  return fetch(url, { method, headers });
}

// React component with defaults in props destructuring
function Pagination({ page = 1, perPage = 10, onPageChange = () => {} }) {
  return <div>Page {page}</div>;
}
5

Common Mistakes

Pitfalls
  • = {} on a destructured object param is required: without it, calling connect() with no args throws because you can't destructure undefined
  • Defaults are evaluated at call time, not definition time — good for objects, avoids the Python mutable-default bug
  • Don't use defaults for required parameters — it hides missing data errors

    }))

04

Rest & Spread Operators

1

Definition

Both use the ... syntax but serve opposite purposes:

  • Spread — expands an iterable (array/object) into individual elements
  • Rest — collects remaining elements into an array or object
3

Syntax & Core Concept

JSSpread — Expanding
// Array spread
const a = [1, 2, 3];
const b = [4, 5];
const combined = [...a, ...b];           // [1,2,3,4,5]
const copy = [...a];                       // shallow copy

// Object spread (ES2018)
const user = { name: "Vipin", age: 25 };
const updated = { ...user, age: 26 };     // { name:"Vipin", age:26 }
const merged = { ...defaults, ...overrides }; // later key wins

// Spread into function args
const nums = [3, 1, 4];
Math.max(...nums);                          // 4
JSRest — Collecting
// Rest in function params — must be last
function sum(...numbers) {
  return numbers.reduce((acc, n) => acc + n, 0);
}
sum(1, 2, 3, 4);  // 10

// Rest in destructuring
const [first, ...rest] = [1, 2, 3];   // rest = [2,3]
const { name, ...others } = user;     // others = { age:25 }
4

Practical Examples

JSReact / Redux patterns
// Immutable state update (Redux / useReducer)
function reducer(state, action) {
  switch (action.type) {
    case "UPDATE_USER":
      return { ...state, user: { ...state.user, ...action.payload } };
    case "REMOVE_ITEM":
      return { ...state, items: state.items.filter(i => i.id !== action.id) };
  }
}

// Merging config objects in Node.js
const config = { ...defaultConfig, ...userConfig, ...envConfig };

// HOF that wraps a function — passes all args through
function withLogging(fn) {
  return function(...args) {
    console.log("called with", args);
    return fn(...args);
  };
}
5

Common Mistakes

Pitfalls
  • Spread is a shallow copy — nested objects are still references
  • Rest must be the last parameter: fn(a, ...b, c) is a SyntaxError
  • Spreading huge arrays in hot code is O(n) — prefer concat or push(...arr) for large arrays
  • Object spread only copies own enumerable properties — prototype methods are lost
6

Advanced Insights

Spread vs Object.assign

{ ...obj } and Object.assign({}, obj) both do shallow copies. Key difference: Object.assign triggers setters on the target; spread does not. Spread is preferred for immutable patterns.

Real Project Usage

JSXReact prop forwarding
// Forward all extra props to the underlying element
function Input({ label, className, ...inputProps }) {
  return (
    <div>
      <label>{label}</label>
      <input className={`input ${className}`} {...inputProps} />

05

Enhanced Object Literals

1

Definition

ES6 introduced several shorthand notations for creating object literals: property shorthand, method shorthand, computed property names, and improvements to object method definitions.

3

Syntax & Core Concept

JS
const name = "Vipin";
const age  = 25;

// 1. Property shorthand — if var name matches key name
const user = { name, age };          // { name: "Vipin", age: 25 }

// 2. Method shorthand — drop "function" keyword
const api = {
  fetch() { return "data"; },       // same as fetch: function() {}
  async load() { ... },
  *generate() { ... }                // generator method
};

// 3. Computed property names — dynamic keys with []
const field = "email";
const form = {
  [field]: "vipin@example.com",
  [`get_${field}`]() { return this[field]; }
};
4

Practical Examples

JSReal World
// Dynamic Redux action creators
const createAction = (type) => (payload) => ({ type, payload });

// Dynamic form state with computed keys
const handleChange = (e) => {
  setForm(prev => ({ ...prev, [e.target.name]: e.target.value }));
};

// Factory function — shorthand makes it tighter
function createUser(name, role) {
  return {
    name,
    role,
    greet() { return `Hi, I'm ${this.name}`; }
  };
}
5

Common Mistakes

Pitfalls
  • Method shorthand creates methods that have their own this — don't confuse with arrow functions (which don't)
  • Computed keys are evaluated at object creation time — they're not reactive
7

06

Iterators & Generators

1

Definition

An iterator is any object with a next() method that returns { value, done }. An iterable is an object with a [Symbol.iterator]() method that returns an iterator. A generator is a function that produces a lazy sequence of values using yield.

2

Why It Exists

Before generators, producing lazy/infinite sequences required manual state machines with closures. Generators provide a clean, pauseable execution model — the foundation of async/await internally.

3

Syntax & Core Concept

JSIterator Protocol
// Manual iterator
function range(start, end) {
  let current = start;
  return {
    next() {
      return current <= end
        ? { value: current++, done: false }
        : { value: undefined, done: true };
    },
    [Symbol.iterator]() { return this; }
  };
}

for (const n of range(1, 3)) console.log(n); // 1, 2, 3
JSGenerator Function
// function* — the * makes it a generator
function* range(start, end) {
  for (let i = start; i <= end; i++) {
    yield i;  // pauses here, returns i, resumes on next .next()
  }
}

const gen = range(1, 3);
gen.next(); // { value: 1, done: false }
gen.next(); // { value: 2, done: false }
gen.next(); // { value: 3, done: false }
gen.next(); // { value: undefined, done: true }

// Spread and for...of work because generators are iterables
[...range(1, 5)]; // [1,2,3,4,5]

// Infinite generator — lazy, won't blow the stack
function* ids() {
  let id = 1;
  while (true) yield id++;
}
const nextId = ids();
nextId.next().value; // 1
nextId.next().value; // 2

// yield* — delegate to another iterable
function* combined() {
  yield* [1, 2];
  yield* [3, 4];
}
4

Practical Examples

JSPagination generator
async function* fetchAllPages(url) {
  let page = 1;
  while (true) {
    const res = await fetch(`${url}?page=${page}`);
    const { data, hasMore } = await res.json();
    yield data;
    if (!hasMore) break;
    page++;
  }
}

for await (const page of fetchAllPages("/api/users")) {
  processUsers(page);
}
5

Common Mistakes

Pitfalls
  • Generators are consumed once — you can't restart them; create a new generator instance
  • Spreading an infinite generator ([...ids()]) will hang the process
  • return in a generator sets done: true and the value is not picked up by for...of
6

Advanced Insights

Generators power async/await

Under the hood, Babel transpiles async/await to generator functions with a runtime scheduler. Each await is a yield point. Understanding generators demystifies the async model deeply.

7

Map, Set, WeakMap, WeakSet

1

Definition

Map is a key-value collection where any value (including objects) can be a key, with guaranteed insertion-order iteration. Set is a collection of unique values. Weak variants hold object references weakly — they don't prevent garbage collection.

3

Syntax & Core Concept

JSMap
const map = new Map();
map.set("name", "Vipin");
map.set(42, "answer");          // number key

const obj = {};
map.set(obj, "meta");           // object as key!

map.get("name");                  // "Vipin"
map.size;                         // 3
map.has(42);                       // true
map.delete(42);

// Initialization from entries
const scores = new Map([["Alice", 95], ["Bob", 87]]);

// Iteration — guaranteed insertion order
for (const [key, value] of scores) {
  console.log(`${key}: ${value}`);
}
JSSet
const set = new Set([1, 2, 2, 3, 3]);
console.log([...set]);  // [1, 2, 3] — duplicates removed

set.add(4);
set.has(2);   // true — O(1) lookup
set.size;     // 4

// Remove duplicates from array — common pattern
const unique = [...new Set(array)];
JSWeakMap / WeakSet
const cache = new WeakMap();

function process(element) {
  if (cache.has(element)) return cache.get(element);
  const result = expensiveCompute(element);
  cache.set(element, result);   // auto-cleaned when element is GC'd
  return result;
}

// WeakSet — track seen objects without preventing GC
const seen = new WeakSet();
function visitOnce(node) {
  if (seen.has(node)) return;
  seen.add(node);
  // process node...
}
4

Map vs Object — When to use which

FeatureObjectMap
Key typesString / Symbol onlyAny value
Insertion orderNot guaranteed (numbers first)Always guaranteed
SizeManual Object.keys().length.size property
Prototype pollutionRisk from inherited keysNo prototype
JSON serializationNativeNeeds manual conversion
Performance (frequent add/delete)SlowerOptimized
5

Common Mistakes

Pitfalls
  • Using Map when a plain object is fine — adds API overhead for no benefit with string keys
  • WeakMap/WeakSet keys must be objects — primitives throw TypeError
  • WeakMap is not iterable (by design) — you can't list all keys
  • Set uses SameValueZero comparison — NaN === NaN in a Set (unlike normal JS)
7

08

Symbol

1

Definition

Symbol is a primitive type (like string or number) whose every instance is globally unique. Symbols are primarily used as unique object property keys that won't accidentally collide with other keys.

2

Why It Exists

Before ES6, libraries had to use prefixed string keys like "_myLib_internal" to avoid collision. Symbols guarantee uniqueness without any naming convention.

3

Syntax & Core Concept

JS
// Symbol() — always unique
const id1 = Symbol("id");
const id2 = Symbol("id");
console.log(id1 === id2); // false — always unique!

// Use as object key — not enumerable by default
const ID = Symbol("id");
const user = { [ID]: 42, name: "Vipin" };

Object.keys(user);       // ["name"] — Symbol not included!
user[ID];               // 42 — accessed via symbol reference

// Symbol.for — global registry (shared across modules)
const shared = Symbol.for("app.user");
Symbol.for("app.user") === shared; // true

// Well-known Symbols — customize built-in JS behavior
class Range {
  constructor(from, to) { this.from = from; this.to = to; }

  [Symbol.iterator]() {   // makes Range iterable!
    let current = this.from;
    const end = this.to;
    return {
      next() {
        return current <= end
          ? { value: current++, done: false }
          : { done: true };
      }
    };
  }
}

[...new Range(1, 5)]; // [1,2,3,4,5]
4

Well-Known Symbols

SymbolPurpose
Symbol.iteratorMake object iterable with for...of
Symbol.asyncIteratorMake object async iterable with for await...of
Symbol.toPrimitiveControl type coercion (e.g. obj + 1)
Symbol.toStringTagCustom [object MyClass] tag
Symbol.hasInstanceCustomize instanceof behavior
5

Common Mistakes

Pitfalls
  • Symbol properties are skipped by JSON.stringify and for...in — use Object.getOwnPropertySymbols() to access them
  • Don't use Symbol() when you need cross-module sharing — use Symbol.for()
  • Symbols can't be converted to string implicitly — "id: " + symbol throws; use symbol.toString()
7

09

for...of vs for...in

1

Definition

  • for...of — iterates over values of any iterable (array, string, Map, Set, generator)
  • for...in — iterates over enumerable property keys of an object (including inherited ones)
3

Syntax & Core Concept

JS
const arr = ["a", "b", "c"];

// for...of — values
for (const val of arr) {
  console.log(val);  // "a", "b", "c"
}

// for...in — enumerable keys (strings!)
for (const key in arr) {
  console.log(key);  // "0", "1", "2" — indices as strings!
}

// for...of works on any iterable
for (const char of "hello") { /* h, e, l, l, o */ }

for (const [key, val] of map) { /* Map entries */ }

// for...in enumerates prototype chain too!
function Animal(name) { this.name = name; }
Animal.prototype.type = "mammal";
const dog = new Animal("Rex");

for (const k in dog) console.log(k); // "name", "type" — prototype included!

// Guard with hasOwnProperty
for (const k in dog) {
  if (Object.prototype.hasOwnProperty.call(dog, k)) {
    console.log(k);  // "name" only
  }
}
4

Comparison Table

Featurefor...offor...in
Iterates overValuesEnumerable keys (strings)
Works onAny iterable (arrays, strings, maps, sets, generators)Any object
Prototype chainNoYes — includes inherited keys
Use for arrays?Yes — preferredAvoid — unexpected behavior
Use for objects?No (unless iterable)Yes — with hasOwnProperty guard
Break / ContinueYesYes
5

Common Mistakes

Pitfalls
  • Using for...in on arrays — you get string indices and may get prototype properties from libraries
  • Using for...of on plain objects — throws because {} is not iterable (use Object.entries())
  • Forgetting that for...of with a Map gives [key, value] pairs — use destructuring
JSCorrect patterns
// Iterate object keys safely
for (const [key, val] of Object.entries(obj)) { ... }

// Iterate with index
for (const [i, val] of arr.entries()) { ... }
7

Optional Chaining (?.)

1

Definition

Optional chaining (?.) lets you safely access deeply nested properties without throwing if an intermediate value is null or undefined. It short-circuits and returns undefined instead of throwing a TypeError.

2

Why It Exists

JSBefore
// Defensive programming hell
const city = user && user.address && user.address.city;
const zip  = user && user.address && user.address.zip || "N/A";
3

Syntax & Core Concept

JS
const user = { profile: { address: { city: "Delhi" } } };

// Property access
user?.profile?.address?.city;     // "Delhi"
user?.profile?.phone?.number;     // undefined (no throw)

// Optional method call
user.getRole?.();                 // undefined if getRole doesn't exist

// Optional bracket access
user?.roles?.[0];                 // first role or undefined

// Combined with nullish coalescing for fallback
const city = user?.address?.city ?? "Unknown";

// Optional call for callbacks
function run(callback) {
  callback?.();  // safe — no need to check if callback exists
}
4

Practical Examples

JSXReact
// API response access
const { data } = useFetch("/api/user");
const avatarUrl = data?.user?.avatar?.url ?? "/default-avatar.png";

// Event handlers — DOM may be null
const handleClick = (e) => {
  e?.preventDefault?.();
  e?.stopPropagation?.();
};

// Next.js server component
const session = await getSession();
const userName = session?.user?.name;
5

Common Mistakes

Pitfalls
  • Overusing ?. everywhere hides bugs — if a property should always exist, don't use optional chaining; let it fail loudly
  • Can't use on the left side of assignment: user?.name = "x" is a SyntaxError
  • Short-circuits the entire chain, not just the failing part
  • obj?.method() calls method only if obj is not null/undefined — but the call itself can still throw if method throws
6

Advanced Insights

Performance

Each ?. adds a null check at runtime. In tight loops over millions of iterations, this overhead accumulates. For hot paths where you know the object exists, use regular property access.

7

Practice Task

Easy

8

JSZustand middleware
const authMiddleware = (store) => (next) => (action) => {
  const token = store.getState()?.auth?.token;
  if (!token) return redirect("/login");
  return next(action);

11

Nullish Coalescing (??)

1

Definition

The nullish coalescing operator (??) returns the right-hand side value only when the left-hand side is null or undefined. Unlike ||, it does not treat 0, "", or false as nullish.

2

Why It Exists — The || Bug

JSThe problem with ||
const port = userConfig.port || 3000;
// Bug: if user sets port=0, this returns 3000 instead of 0!
// 0 is falsy, so || replaces it

const label = input.label || "Default";
// Bug: if label="" (intentionally empty), this replaces it
3

Syntax & Core Concept

JS
// ?? only triggers on null or undefined
const port  = userConfig.port ?? 3000;     // 0 → 0 (correct!)
const label = input.label ?? "Default";   // "" → "" (correct!)
const flag  = options.active ?? true;     // false → false (correct!)

// null → fallback
const name = null ?? "Guest";    // "Guest"
// undefined → fallback
const role = undefined ?? "user"; // "user"

// Nullish assignment operator ??= (ES2021)
user.name ??= "Anonymous"; // assigns only if null/undefined

Common Mistakes

Pitfalls
  • Mixing || and ?? without parens is a SyntaxError: a || b ?? c — use (a || b) ?? c
  • Using ?? when you actually want a falsy guard — e.g., you do want 0 to trigger the fallback
  • Confusing ??= (nullish assign) with ||= (OR assign) — they behave differently on falsy values

Advanced Insights

Logical Assignment Operators (ES2021)
  • x ??= y — assign y only if x is null/undefined
  • x ||= y — assign y only if x is falsy
  • x &&= y — assign y only if x is truthy
